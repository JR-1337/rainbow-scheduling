## Key Context

- **Push status:** Commits `fe536fa`, `8ace121`, `54dd301` are local only. Push before demo rehearsal so Vercel builds the live URL Sarvi + owner will see.
- **Density toggle CSS is broad:** `.density-compact .grid > div { padding: 2px !important; }` - the `.grid` selector catches any Tailwind grid inside the density container. Works for the admin schedule but monitor for regressions if other grids nest inside.
- **Bottom nav active state** is derived, not stored: `mobileMenuOpen ? 'more' : mobileAnnouncementOpen ? 'alerts' : requestModalOpen ? 'requests' : 'schedule'`. Closing any modal automatically snaps active back to 'schedule'.
- **Redundancy removed:** mobile employee no longer has a hamburger or bell in the header; mobile admin no longer has a hamburger or Requests/Comms tabs in the filing tabs row. All those destinations now reach via bottom nav. Sarvi has to learn the new layout - demo rehearsal should cover it.

- **AnimatedNumber** is now decimal-aware (`decimals` prop). Uses `Math.round(x * 10^d) / 10^d` to avoid float drift.
- **Phase 10 is partial:** only employee shift detail is in the bottom sheet. Admin quick approve/deny + time off details are still regular modals. Post-demo work.
- **Phase 9 density toggle:** persists via `localStorage.getItem('otr-density')`. Default 'comfortable'. Toggle lives next to the Settings gear in admin desktop header.
