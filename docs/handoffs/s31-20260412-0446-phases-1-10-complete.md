## Key Context

- **Push is deferred to JR**: Vercel auto-deploys on push and the live URL is Sarvi's demo environment. JR wanted to eyeball the diff before the URL updates. `git push origin main` when ready.
- **Perf helpers at module level**: `ROLES_BY_ID` (O(1) role lookup) and `toDateKey(date)` (no ISO alloc) are exported from App.jsx. Both mobile files already import them. Use these, not `ROLES.find()` or `.toISOString().split('T')[0]`.
- **Precomputed date arrays**: In admin desktop branch, `currentDateStrs`, `allDateStrs`, `todayStr` are memoized. Any new loop over dates should use these, not re-run `.toISOString()`.
- **Memo is only effective with stable handler refs**: `handleCellClick`, `handleEditEmployee`, `handleShowTooltip`, `handleHideTooltip` are all useCallback. Any new inline arrow in an EmployeeRow prop breaks memo — add to the useCallback set instead.
- **Bottom nav active state derived, not stored**: `mobileMenuOpen ? 'more' : mobileAnnouncementOpen ? 'alerts' : ...`. Closing any modal auto-snaps back to 'schedule'. No new state field needed.
- **Density toggle CSS scope is broad**: `.density-compact .grid > div { padding: 2px !important; }` catches any Tailwind grid inside the compact container. Works fine for the admin schedule. If a new nested grid appears inside, it'll also compact — watch for this.
- **Shadows = accent halos**: `THEME.shadow.card`/`cardSm` use rotating OTR accent (red/blue/orange/green/purple) not dark drop-shadows. Green rotation is the one to eyeball in demo — may feel too bright.
- **Phase 10 is partial**: only employee shift detail is in a bottom sheet. Admin quick approve/deny + time-off details still use full modals. Post-demo work.
- **Mobile header is LEAN**: no hamburger, no bell. All modal destinations reach via bottom nav. Filing tabs (Wk1/Wk2/Mine) only render on schedule/mine destinations. Sarvi learns this at demo rehearsal.
- **AnimatedNumber rebuilt**: now takes `decimals`, `suffix`, `overtimeThreshold` props. Hours render as `12.5h` not rounded `13`.

## Anti-Patterns (Don't Retry)

- **Light mode with OTR terracotta accent** (since S28) — JR rejected
- **Gradient background blobs** (since S28) — cards looked pasted on
- **Transparent accent-tinted card backgrounds** (since S28) — invisible on dark
- **Changing OTR accent colors** (since S29) — IMMUTABLE: Red #EC3228, Blue #0453A3, Orange #F57F20, Green #00A84D, Purple #932378
- **"Too many colors" complaints for Rainbow brand** (since S29) — the brand IS multiple colors
- **Muting/desaturating role colors** (since S29) — role colors stay saturated; only functional status indicators get desaturated
- **Stopping mid-execution to ask direction** (since S30) — plan approved + build green = keep going
- **Dark drop-shadows on dark bg** (since S31) — use accent-color halos (see `THEME.shadow`)
- **Duplicate entry points for same destination on mobile** (since S31) — one canonical path; bottom nav owns drawer/requests/alerts
- **Inline arrow functions as grid row props** (since S31) — breaks React.memo on EmployeeRow; use the existing useCallback handlers
- **`.toISOString().split('T')[0]` in hot paths** (since S31) — use `toDateKey(date)` (~10x faster, no ISO alloc)
