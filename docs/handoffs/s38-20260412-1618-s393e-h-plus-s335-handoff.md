## Anti-Patterns (Don't Retry)

- **Deleting the inline component block from App.jsx BEFORE adding the import for the extracted version** (new in S39.3g, caused mid-session crash) — always: (1) write the extracted file, (2) add the import to App.jsx, (3) delete the inline block, (4) build + preview, (5) commit. If the order slips, the crash won't show up in `npm run build` (vite doesn't catch undefined JSX refs) but it WILL ReferenceError at runtime when the code path executes. Caught only because session-start protocol forced a fresh grep + verify.
- **Executing S39.4 per the plan file** (since S36) — conflicts with decisions.md 2026-02-10. `isMobileAdmin` branch reads 30+ App-scope state vars via closure. Revisit only after state refactor.
- **Asking JR "which plan step next?"** (since S35) — the plan is the answer. Exception: flagging a genuine decisions.md conflict.
- **Trusting `result.success` from `chunkedBatchSave`** (since S34) — callers must read `data.{totalChunks, failedChunks}` on `success: false`.
- **Top-level use of imported symbols in `src/pdf/generate.js` / `src/email/build.js` / `src/panels/*.jsx` / `src/modals/*.jsx`** (since S34) — circular imports with `../App` are safe ONLY inside function bodies.
- **Listing post-demo deferrals in next-task options** (new this session, saved as `feedback_no_post_demo_reminders.md`) — until 2026-04-14 meeting, drop deferred items from choice menus entirely.
- **Using `git add -A` or `git add .`** — always explicit paths. `Photos/` and `dist/` are now gitignored so lower risk, but habit still matters.
