# How to turn on the new login security

You only have to do this once. Takes about 5 minutes.

Do these three steps **in order**. If anything looks different from what's written here, stop and ask — don't guess.

---

## Step 1 — Paste the new backend code

1. On your computer, open the file `backend/Code.gs` (in the project folder).
2. Select everything in it (**Ctrl+A**) and copy (**Ctrl+C**).
3. Open [script.google.com](https://script.google.com) and open the **Rainbow Scheduling** project.
4. Click the **< >** icon on the left (Editor).
5. Click the file called `Code.gs`.
6. Click inside the code area, select everything (**Ctrl+A**), paste (**Ctrl+V**). The old code is replaced with the new code.
7. Click the **Save** icon at the top (or **Ctrl+S**). Wait for "Saved."

---

## Step 2 — Run the one-click setup

This adds the two new sheet columns and generates the security secret for you.

1. Open the Google Sheet that the app reads from.
2. **Close the tab and open it again.** (This is needed so the new menu shows up.)
3. At the top you'll see a new menu called **Rainbow Admin** (next to File / Edit / View).
4. Click **Rainbow Admin → Run Auth Setup (one-time)**.
5. The first time only, Google will ask you to authorize the script. Click through: **Review permissions → choose your account → Advanced → Go to (unsafe) → Allow.** This is Google being cautious because the script is not "verified" — it's your own script, so it's fine.
6. Re-run **Rainbow Admin → Run Auth Setup (one-time)** if the auth prompt interrupted it.
7. A box appears listing what it did (added columns, generated secret). Click **OK**.

You can re-run this button — it will say "already exists" for anything already done, so it's safe. But you shouldn't need to.

---

## Step 3 — Push it live

1. Go back to the Apps Script editor tab.
2. Click the blue **Deploy** button at the top right → **Manage deployments**.
3. Find the existing deployment in the list, click the **pencil (Edit)** icon next to it.
4. In the **Version** dropdown, choose **New version**.
5. In the description box, type: `S36 auth rebuild`
6. Click **Deploy**. Wait until it says "Deployment successfully updated."
7. Close the dialog.

**Do not** create a new deployment — use Edit on the existing one so the app keeps its URL.

---

## How to check it worked

1. Go to https://rainbow-scheduling.vercel.app
2. Log in with your normal email + password.
3. If the login works and you see the schedule, you're done. The server quietly upgraded your password to the secure format during that login.
4. Log out and log back in — should still work.

### When you've confirmed it works

Tell Claude: **"Setup confirmed, please strip the setup menu (v2.15)."**

Claude will remove the `Run Auth Setup` button from Code.gs so you can't hit it by accident later. The **Rotate HMAC Secret** emergency button will stay.

---

## If login fails after deploy

- Most likely cause: the secret didn't save. Open Apps Script → gear icon (Project Settings) → Script Properties. You should see a row called `HMAC_SECRET` with a long random value. If it's missing, re-run **Rainbow Admin → Run Auth Setup**.
- Second cause: you pasted the code but forgot to press Save before deploying. Go back to Step 1.

---

## Emergency — force everyone to log out

Click **Rainbow Admin → Rotate HMAC Secret**. Confirm. Every logged-in user (including you) is bounced to the login screen on their next click. Use this if you think a token has been stolen or after firing someone who had app access.

---

## What NOT to do

- Don't share the `HMAC_SECRET` value with anyone. Treat it like a master password.
- Don't delete the `password`, `passwordHash`, or `passwordSalt` columns from the Employees sheet.
- Don't edit the values in `passwordHash` or `passwordSalt` by hand. They're generated by the app.
